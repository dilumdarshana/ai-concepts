# RAG with HuggingFace Embeddings + Pinecone + LangChain

Express.js API that demonstrates a retrieval-augmented generation (RAG) pipeline using HuggingFace embeddings, Pinecone vector store, and OpenAI GPT-4o.

## How it works

1. **POST /embeddings** — generate 384-dimensional embeddings via HuggingFace's free `all-MiniLM-L12-v2` model and upsert into a Pinecone serverless index
2. **POST /query** — embed a query, search Pinecone for nearest neighbors, optionally filter by year (e.g. `"movies from 2010"`)
3. **POST /chat** — same retrieval step, then feed context into GPT-4o via a LangChain prompt chain to generate an answer
4. **POST /create-index** — create a serverless Pinecone index (aws, us-east-1)

## Prerequisites

- Node.js 24.15 (see `.nvmrc`)
- pnpm 11.5+
- Pinecone account (free tier)
- HuggingFace token (free tier)
- OpenAI API key

## Setup

```sh
cp .env_example .env
# fill in PINECONE_API_KEY, HUGGINGFACE_TOKEN, OPENAI_API_KEY
pnpm install
pnpm dev:rag-huggingface
```

Server starts on `http://localhost:3000`.

## API

### `POST /create-index`

```json
{ "name": "my-index" }
```

### `POST /embeddings`

```json
{
  "text": "Movie plot text to embed...",
  "metadata": { "name": "Movie Title", "year": 2010 }
}
```

### `POST /query`

```json
{ "query": "What movies are from 2010?" }
```

Returns top 5 matches with metadata. Automatically filters by year if a 4-digit year is found in the query.

### `POST /chat`

```json
{ "query": "What is the movie featured by Dilum?" }
```

Returns an answer generated by GPT-4o using retrieved context.

## Tech stack

| Component | Technology |
|---|---|
| Embeddings | HuggingFace `sentence-transformers/all-MiniLM-L12-v2` (384d) |
| Vector store | Pinecone serverless (aws, us-east-1) |
| LLM | OpenAI GPT-4o |
| Orchestration | LangChain `RunnableSequence` |
| Runtime | Node.js, Express, TypeScript |

## Metadata filtering

Queries containing a 4-digit year (e.g. `"movies in 2010"`) automatically apply a Pinecone metadata filter on the `year` field. This ensures vector search only returns records matching that year, improving accuracy over pure semantic search.
